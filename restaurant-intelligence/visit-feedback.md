# 방문 후기 관리

최신 업데이트: 2026-05-18

## 목적

이 파일은 실제 방문 후기를 기록하고, 그 결과를 맛집러 신뢰도와 사용자 취향 모델에 반영하기 위한 운영 문서입니다.

역할은 분리합니다.

- 맛집러 파일은 “누구를 어떤 분야에서 믿을 것인가?”를 답합니다.
- 취향 파일은 “나는 어떤 식당과 메뉴를 좋아하는가?”를 답합니다.
- 방문 후기는 실제 식사 경험을 통해 두 파일을 업데이트하는 근거가 됩니다.

## 업데이트 흐름

1. 방문 결과를 이 파일에 기록합니다.
2. 식사 경험이 개인 취향, 비선호, 가격 민감도, 메뉴 선호, 장르 선호를 드러내면 `taste-profile.md`를 업데이트합니다.
3. 특정 장르의 판단 기준이 바뀌면 `category-analysis.md`를 업데이트합니다.
4. 추천한 맛집러의 신뢰도가 올라가거나 내려가면 `creator-registry.md`를 업데이트합니다.
5. 일반 평가 기준 자체가 바뀔 정도의 피드백이면 `trust-rubric.md`를 업데이트합니다.
6. 비슷한 식당을 올리거나, 보류하거나, 제거해야 하면 `recommendation-queue.md`를 업데이트합니다.
7. 변경한 파일은 `daily-update-log.md`에 기록합니다.

## 평가 반영 기준

| 후기 신호 | 업데이트 |
|---|---|
| 방문이 명확한 hit이고 맛집러의 추천 이유와 실제 경험이 일치함 | 해당 장르에서 맛집러 신뢰도를 올리고, 일치한 취향 신호를 `taste-profile.md`에 추가합니다. |
| 음식은 좋았지만 맛집러의 추천 이유가 모호하거나 빗나감 | 식당 평가는 좋게 남기되, 맛집러 신뢰도는 크게 올리지 않습니다. |
| 강한 추천을 믿고 갔는데 실망함 | 해당 장르/가격대에서 맛집러 신뢰도를 낮추고, 부정 취향 신호를 추가합니다. |
| 비싸지만 퀄리티가 명확히 납득됨 | 비싼 식당이어도 긍정 가성비 신호로 기록합니다. |
| 가격 대비 장인성, 양, 경험값이 부족함 | 부정 가성비 신호로 기록하고 비슷한 후보를 낮춥니다. |
| 특정 메뉴만 좋고 식당 전체는 애매함 | 식당 전체를 과대평가하지 않고 메뉴 단위 hit로 기록합니다. |

## 현재 사용자 후기 반영 원칙

- 가장 강한 긍정 신호는 분야 최고, 국내 1위, 장인의 퀄리티, 진짜 퀄리티가 받쳐주는 극강의 가성비, 인생 맛집, 단골 맛집, 꼭 경험해볼 만한 식사입니다.
- 맛집러 신뢰도와 취향 적합성은 분리합니다. 믿는 맛집러가 추천해도 내 취향이 아닐 수 있고, 좋은 식당이어도 추천자의 전체 신뢰도는 아직 검증 전일 수 있습니다.
- 한 번의 좋은 방문만으로 맛집러를 전면 승격하지 않습니다. 단, 맛집러가 말한 정확한 추천 이유와 실제 경험이 일치하면 해당 장르 신뢰도는 올릴 수 있습니다.
- 한 번의 실패만으로 맛집러를 전면 강등하지 않습니다. 단, 숨은 협찬, 피상적 바이럴, 주문 디테일 부족 같은 반복 가능한 문제가 보이면 장르별로 낮춥니다.
